# ◼ Monolith — Structural English Visualizer

> 映画『2001年宇宙の旅』において、モノリスが「言葉」で何かを語ることはなかった。
> しかし、その存在そのものが強力なメッセージとして機能した。

**文法用語を使わずに、英語の構造を「見える」ようにする。**

Monolithは、英文をSVO構造に基づいて色分けされたネストブロックで可視化するWebアプリケーション。AIが構文解析を行い、学習者は分析する側ではなく、構造を「体験する」側に回れる。

![Monolith Screenshot](docs/screenshot.png)

## コンセプト

従来の英語学習は「主語」「関係代名詞」といった**文法用語（言語化されたラベル）**で文法を説明する。しかし、記号で記号を説明するこのアプローチは、すべての学習者にとって最適とは限らない。

Monolithは**色と空間配置**だけで英語の構造を伝える：

- 🔵 **Subject** — 主語
- 🔴 **Verb** — 動詞
- 🟢 **Object** — 目的語
- 🟡 **Complement** — 補語
- 🟣 **Modifier** — 修飾語句
- 🟠 **Relative clause** — 関係詞節
- 🩵 **Subordinate clause** — 従属節
- ⚪ **Conjunction** — 接続詞

節の入れ子はブロックのネストで表現される。文法用語を覚えなくても、**色のパターンを繰り返し見る**ことで、英語の構造が身体的に身についていく。

## 記号創発としての英語学習

Monolithのアプローチは、[記号創発システム論](https://en.wikipedia.org/wiki/Symbol_emergence_in_robotics)の知見に基づいている。

従来の英語教育:
```
記号（文法用語）→ 記号（英語）= 記号間マッピング
```

Monolithのアプローチ:
```
身体的経験（視覚パターン）→ 意味（構文構造）= 直接接地
```

色とブロックという**新しい記号体系**を通じて、英語の構造が身体的（視覚的）に立ち上がる。文法用語（既存の記号）を経由せず、視覚パターンから直接構造を理解する——これは記号接地問題（Symbol Grounding Problem）への実践的アプローチでもある。

## AGI時代になぜ英語を学ぶのか

AI翻訳で表面的なコミュニケーションは済む時代に、なぜ英語の構造を理解する必要があるのか。

**言語構造が異なるということは、思考の組み立て方が異なるということ。**

- 日本語（SOV + 助詞）: 要素の関係性を後から修飾して構築
- 英語（SVO + 語順）: 配置そのものが意味を決定

新しい言語構造を獲得することは、新しい概念空間を手に入れること。翻訳AIでは代替できない認知的な拡張がそこにある。

## 機能

### 構文解析＆ビジュアライズ
英文を入力すると、AIが構文解析を行い、色分けされたネストブロックで表示。

### 和訳ツールチップ
ブロックにマウスオーバーで、そのブロックの日本語訳を表示。

### 単語モード（Shift）
Shiftキーを押しながらホバーで、単語ごとの文脈に応じた日本語訳を表示。

### テキスト読み上げ（TTS）
- ブロック内テキストをクリック → ブロック全体を発音
- Shift + クリック → その単語だけを発音
- Web Speech API使用（無料、APIキー不要）

### 複数文対応
パラグラフをそのまま入力すると、文ごとに分けて解析・表示。

## 使い方

1. 英文を入力してEnter（またはParseボタン）
2. 色分けされたブロック構造を**眺める**
3. 気になるブロックにマウスオーバーで和訳を確認
4. Shiftを押しながらホバーで単語単位の訳を確認
5. クリックで発音を聞く

**コツ**: 文法を「分析」しようとしないこと。色のパターンを繰り返し見ることで、構造認識が自然に身についていく。

## セットアップ

### 必要なもの
- Python 3.9+
- Gemini APIキー（[Google AI Studio](https://aistudio.google.com/apikey)で無料取得）

### インストール

```bash
pip install flask google-genai
```

### APIキー設定

```bash
mkdir -p ~/.config/google
echo "YOUR_API_KEY" > ~/.config/google/gemini_api_key
```

### 起動

```bash
python3 app.py
# → http://localhost:8793
```

## 名前の由来

映画『2001年宇宙の旅』のモノリス。

言葉で語ることなく、存在そのものが強力なメッセージとして機能した黒い直方体。Monolithも同じように、文法用語という「言葉」を使わずに、色とブロックという**存在そのもの**で言語構造を伝える。

そして「モノリス」は「一枚岩＝ブロック」。ブロックで構造を可視化するこのツールの名前として、完璧に一致している。（これは命名時には意図していなかった偶然の一致。）
