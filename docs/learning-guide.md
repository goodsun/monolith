# Monolith 学習ガイド

## はじめに

Monolithは「勉強する」ツールではなく「体験する」ツールです。文法を分析しようとせず、色のパターンを眺めることから始めてください。

## 基本操作

| 操作 | 通常モード | Shiftモード |
|------|-----------|------------|
| ホバー | ブロック全体の和訳 | 単語ごとの和訳 |
| クリック | ブロック全体を発音 | その単語だけ発音 |

## 色の意味

文法用語は覚えなくてOK。色のイメージだけ持ってください：

- 🔵 **青** — 「誰が」「何が」（動作の主体）
- 🔴 **赤** — 「する」（動作そのもの）
- 🟢 **緑** — 「何を」（動作の対象）
- 🟡 **黄** — 「＝何」（イコールの先）
- 🟣 **紫** — 「いつ・どこで・どうやって」（追加情報）
- 🟠 **橙** — 「〜するところの」（説明を追加する文）
- 🩵 **青緑** — 「〜ということ」（文の中の文）
- ⚪ **灰** — 「が・を・に」のような接着剤

## ステップ1: シンプルな文で色を覚える

まずは3〜5語の短い文から：

```
The cat sat on the mat.
→ 🔵[The cat] 🔴[sat] 🟣[on the mat]

She is a doctor.
→ 🔵[She] 🔴[is] 🟡[a doctor]

I like coffee.
→ 🔵[I] 🔴[like] 🟢[coffee]
```

**ポイント**: 青→赤→(緑/黄/紫) の流れが英語の基本骨格。日本語の「誰が（青）〜を（緑）〜する（赤）」とは順番が違うことに注目。

## ステップ2: 修飾語句に慣れる

紫ブロックが増えてきます：

```
I ate lunch at the park yesterday.
→ 🔵[I] 🔴[ate] 🟢[lunch] 🟣[at the park] 🟣[yesterday]
```

紫（追加情報）はいくつでも後ろに付け足せる。これが英語の「前から後ろに伸びる」感覚。

## ステップ3: ネストを体験する

ブロックの中にブロックが入る構造：

```
I think that she knows what he said.
→ 🔵[I] 🔴[think] 🩵[ ⚪[that] 🔵[she] 🔴[knows] 🩵[ ⚪[what] 🔵[he] 🔴[said] ] ]
```

外側のブロックが内側のブロックを包んでいる。コードのインデントと同じ感覚。

## ステップ4: 実際の英文を読む

ニュース記事やWikipediaの文をコピペして入力：

1. まず色のパターンを**眺める**（分析しようとしない）
2. 気になるブロックにホバーして和訳を確認
3. Shiftモードで知らない単語の意味を確認
4. クリックして発音を聞く
5. もう一度全体を眺める

## 効果的な学習のコツ

### やること
- ✅ 毎日2〜3文でいいので継続する
- ✅ 同じ構造パターンが出てきたら「またこれか」と思う（それが学習）
- ✅ ネストが深い文は「外側→内側」の順に色を追う
- ✅ 発音機能を使って音と構造を結びつける
- ✅ Shiftモードで未知の単語を文脈で覚える

### やらないこと
- ❌ 文法用語を覚えようとする
- ❌ 「なぜこの色か」を論理的に分析しようとする
- ❌ 一度に大量の文を処理する
- ❌ 完璧に理解しようとする（パターン認識は無意識に進む）

## おすすめ教材

入力する英文の難易度目安：

| レベル | 素材 | 特徴 |
|--------|------|------|
| 初級 | 子供向けニュース（News for Kids） | 短文、基本SVO |
| 中級 | Wikipedia簡易版（Simple English Wikipedia） | 適度なネスト |
| 上級 | BBC / NYT 記事 | 複雑なネスト、長い修飾句 |
| 挑戦 | 学術論文のAbstract | 多重ネスト、専門語 |

## なぜ「見るだけ」で上達するのか

人間の脳はパターン認識に優れています。同じ色の組み合わせを何度も見ることで、意識的に覚えなくても「英語の文はこういう構造をしている」という感覚が形成されます。

これは言語習得の自然なプロセス——子供が文法を学ばずに母語を習得するのと同じ原理——を、視覚的なパターンで加速させるアプローチです。
