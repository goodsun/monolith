# Monolith — コンセプトノート

## 問題意識

### 言語構造の壁

日本語話者が英語を学ぶとき、単語や文法ルールを覚えるだけでは不十分な壁がある。それは**構造の構築方法そのものが異なる**ということ。

- **日本語**: SOV + 助詞。要素の関係性を後から修飾して構造を組み立てる。「後ろから前に」積み上げる
- **英語**: SVO + 語順。配置そのものが意味を決める。「前から後ろに」展開する

```
日本語: 「彼が言ったことを彼女が知っていると私は思う」
         ←←←←←←←←← 内側から外側へ

英語:   "I think that she knows what he said"
         →→→→→→→→→ 外側から内側へ
```

思考の展開方向が真逆。英語を学ぶとは、単語を置き換えることではなく、**思考の組み立て方そのものを別の設計図で再構築する**こと。

### 文法用語の罠

従来の英語教育は「主語」「関係代名詞」「分詞構文」といった文法用語で文法を説明する。しかし、これは**記号で記号を説明する**というアプローチであり、本質的に：

1. 文法用語を日本語で覚える
2. その用語と英語の構造を対応づける
3. 英文を見たときに用語を思い出す
4. 用語の定義に基づいて構造を理解する

という**多段階の間接参照**を要求する。

特に視覚・空間的な認知（PRIなど）が得意な学習者にとって、この言語ラベルベースのアプローチは認知特性と合致しない。

## 解決策: 視覚的構造接地

### 色で語る

MonolithはSVO構造を色で表現する：

| 色 | 役割 | 従来の文法用語 |
|----|------|---------------|
| 🔵 青 | 動作の主体 | 主語 |
| 🔴 赤 | 動作そのもの | 動詞 |
| 🟢 緑 | 動作の対象 | 目的語 |
| 🟡 黄 | 状態の説明 | 補語 |
| 🟣 紫 | 追加情報 | 修飾語句 |
| 🟠 橙 | 関係する追加文 | 関係詞節 |
| 🩵 青緑 | 従属する文 | 従属節 |
| ⚪ 灰 | つなぎ言葉 | 接続詞 |

学習者は文法用語を**一切覚える必要がない**。色のパターンを繰り返し見ることで、英語の構造認識が自然に形成される。

### ネストで階層を可視化

英文の複雑さの多くは**節の入れ子**に起因する。Monolithはこれをブロックのネストで表現する：

```
┌─ 青 [I] ─┐ ┌─ 赤 [think] ─┐ ┌─────── 青緑 ──────────────────┐
│           │ │              │ │ ⚪[that] 青[she] 赤[knows]   │
│           │ │              │ │  ┌─── 青緑 ──────────┐       │
│           │ │              │ │  │ ⚪[what] 青[he]   │       │
│           │ │              │ │  │ 赤[said]          │       │
│           │ │              │ │  └────────────────────┘       │
└───────────┘ └──────────────┘ └───────────────────────────────┘
```

コードのネスト、DOMツリー、ディレクトリ構造——エンジニアにとっては馴染みのある表現。英語の文構造もこれと同じ視点で捉えられる。

## 理論的背景: 記号創発

### 記号接地問題

人工知能研究において「記号接地問題（Symbol Grounding Problem）」は古典的な難題である。記号（言葉）が実世界の意味とどう結びつくのか——これは言語学習にもそのまま当てはまる。

従来の英語教育は、英語という記号体系を日本語という別の記号体系を経由して学ぶ。これは記号間マッピングであり、真の意味での「接地」が起きにくい。

### 身体的経験を通じた記号の創発

谷口忠大の記号創発システム論が示すように、記号の意味は身体的な経験を通じて創発する。Monolithは：

1. **身体性**（視覚・空間認知）を活用し
2. **新しい記号体系**（色・ブロック・ネスト）を通じて
3. **英語の構造という意味**を直接接地させる

文法用語を経由しない、視覚パターンからの直接的な意味理解。これは記号創発の実践的応用と言える。

### 認知特性への適応

WAIS-IVのPRI（知覚推理）が高い学習者は、視覚・空間的な情報処理に優れる。こうした学習者にとって：

- ❌ 文法用語の暗記 → 言語的処理（WMI/VCI依存）
- ⭕ 色とブロックのパターン認識 → 視覚的処理（PRI活用）

Monolithは認知特性に合った学習チャネルを提供する。これはユニバーサルデザインとしての英語学習の一形態でもある。

## AGI時代における意義

### なぜAIに訳させるだけでは不十分か

AI翻訳の精度が向上した現在、「英語を学ぶ必要があるのか」という問いは正当である。しかし：

**言語構造を理解することは、新しい認知フレームを獲得すること。**

異なる言語構造で思考できることは、同じ対象を異なる角度から捉える能力を与える。これはAIには代替できない、人間の認知そのものの拡張である。

### AIが担う役割

Monolithにおいて、AIは**構文解析という専門的な作業**を担当する。これにより：

- 学習者は文法知識がなくても構造の可視化を利用できる
- 「分析する側」ではなく「体験する側」に回れる
- 学習の入り口のハードルが劇的に下がる

**AIが解析し、人間が体験する。** これがAGI時代の言語学習の一つの形。

## 設計思想

### 引き算のデザイン

Monolithの画面には文法用語が一切表示されない。色と配置だけ。

これは映画のモノリスと同じ哲学——**語らずに伝える**。学習者に「分析しろ」と言うのではなく、見ているだけで構造が浮かび上がる体験を提供する。

### ダークテーマの意図

黒い背景に浮かぶ色付きブロック——これはモノリスの漆黒に浮かぶ光でもあり、夜空に浮かぶ星座でもある。暗い背景は色の知覚を高め、構造のコントラストを際立たせる。
